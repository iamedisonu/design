6 Data And Contract Notes

The current frontend is self-contained. It does not call backend endpoints, and it does not depend on any hidden data model to render the UI you see now.

What is shown visually

  • Workspace names: Circles, Graduate, IRS, Academy.
  • Space icons on the sidebar and settings icon for workspace configuration.
  • Chat menus with pin, rename, archive, and bin actions.
  • Evidence shown as a small card inside assistant answers.

What is not in this frontend

  • No backend endpoints.
  • No identity review screen.
  • No access model or role UI.
  • No login flow in the current build.

Evidence note

Evidence is displayed visually in a compact box so chat answers can show the source context without taking over the page. It is presentational in this frontend-only build.

8 Chat And Evidence

The chat view now behaves like a focused working area: message thread, composer, evidence box, and context-aware menu actions. The evidence block is small on purpose so it supports the answer instead of crowding it.

Chat experience

Users can open chats from spaces, create a new untitled chat, rename it after the first prompt, and keep the current workspace in view.

Evidence style

Evidence is rendered as a compact bordered box with rows for source context so the answer stays readable and visually calm.

9 Visual Direction

The UI is tuned to be calmer in light mode and still readable in dark mode. Colors are used as plates and status cues, not decoration.

Green#8CC63D

Lines, separators, and quiet structure.

Simple cards

Overview and Data Health use small cards with enough detail to be useful and no more.

Workspace colors

Settings includes color plates and look options so the interface can feel different without becoming busy.

Readable light mode

The white theme is softened so content stays visible without harsh brightness.

10 Rules And Guardrails

These are the rules that keep the frontend honest and easy to maintain.

Do

  • +Keep the UI compact and readable.
  • +Use workspace names, icons, and color plates consistently.
  • +Show evidence visually without overwhelming the chat.
  • +Keep Overview to the main counts users need.
  • +Use confirmation for risky bulk actions like clearing chats to bin.

Do Not

  • -Bring backend endpoints into the current frontend.
  • -Reintroduce identity review or access model UI into the current build.
  • -Add dense cards or extra dashboards just to fill space.
  • -Make Data Health feel like a technical control panel.
